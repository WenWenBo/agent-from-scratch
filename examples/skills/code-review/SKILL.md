## 审查维度

请从以下维度审查代码：

1. **安全性**：检查硬编码密钥、SQL 注入、XSS 等安全风险
2. **Bug 风险**：检查空指针、除零、类型错误等潜在 Bug
3. **代码风格**：检查命名规范、函数长度、重复代码等风格问题
4. **性能**：检查不必要的循环、内存泄漏等性能问题

## 输出格式

对每个发现的问题，按以下格式输出：

- **严重级别**：${severity_levels}
- **文件位置**：文件名:行号
- **问题描述**：具体描述
- **修复建议**：如何修复

最后给出总体评分（1-10 分）和总结。
